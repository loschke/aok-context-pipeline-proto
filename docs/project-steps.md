### Bild 3: Schritt 2: Systemfähigkeiten („Motor“)

**Output: Motor-Raum "designt"; Status quo und Leitplanken erfasst**

**To-Do’s**

**Capability Scope erfassen >>**

1. **Content-Capability** (Content-Typen, Metadaten, Struktur, Themenbibliothek >> siehe Content-Audit)

2. **Prozess-Capability** (Digitalisiert/ Digitalisierbar, Zugriff auf Daten möglich, Zugriff/ APIs, Logik?)

3. **Identitäts- & Kontext-Capability** (personenbezogene Nutzerdaten, neue pot. Daten (Health, Smartwatch etc.), Gesundheits/Krankendaten, Authentifizierung, Einstellungen d. Nutzers)

4. **KI-Orchestrierungs-Capability** (Intent verstehen, Kern-Handlungen, Delegation/ Human-the-Loop)

5. **Governance-Capability** (Guidelines, Tests, Monitoring, ...)

**Audits durchführen (Status quo verstehen, Rahmenbedingungen erkunden)**

1. **Architektur- und Daten-Audit** (Systemlandschaft erfassen (Ist vs. Soll), APIs identifizieren, Echtzeit-Zugriffe prüfen, Datenqualität prüfen, Eigentümer der Daten klären, Lücken identifizieren)

2. **Dienstleister-Audit** (wer kann was, wer verantwortet, gibt es Lücken? Verantwortungsüberschneidung)

3. **Content-Audit** (Struktur des Wissens erfassen, Content-Typen identifizieren, Redundanzen prüfen, Aktualitätsprüfung, Lücken identifizieren)

4. **Governance Audit** (Datenschutz, Entscheidungsgrenzen, Human-in-the-loop-Pflichten, Haftungsfragen)
